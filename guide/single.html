1. Introduction

This plugin integrates Google Website Optimizer into Grails applications to make A/B and multivariate testing easier.

It is best used for testing changes in the view layer, as Google Website Optimizer relies heavily on JavaScript.

The plugin does not provide any reporting integration. You would still need to access your google analytics account to see the results of your A/B tests and to create new experiments

2. Getting Started

This plugin wraps the JavaScript code used by Google Website Optimizer (GWO) into Grails tags under the namespace gwo.

Before you start, you need to have a Google Analytics account set up for your site and enable website optimizer by visiting the GWO site.

For a discussion on when to use an AB test and when to use an Multivariate test, please refer to the resources from the further reading.

2.1. Setting up an AB Test

This step assumes that you already have 2 variations of your pages that you wish to test. In Grails terms, these would be two separate actions or views.

Set up an experiment

  • Login to your GWO account and select Experiments > Create another experiment
  • Choose A/B Experiment - The simplest way to start testing fast
  • Run through the checklist and click on "I've completed the steps above checkbox"
  • Click on "Create"
  • Fill in all the fields in the page
  • Click on "Continue"
  • Select 'You will install and validate the JavaScript tags'

Get your account and experiment ids.

At this point, you will see a few JavaScript tags that google requires you to embed on your site.

The two bits of information you want is your account identifier and the experiment identifyer.

Under 2. Variation Pages, you will see a Tracking Script that looks like the following

<!-- Google Website Optimizer Tracking Script -->
<script type="text/javascript">
  var _gaq = _gaq || [];
  _gaq.push(['gwo._setAccount', 'UA-25983867-2']);
  _gaq.push(['gwo._trackPageview', '/0156495775/test']);
  (function() {
    var ga = document.createElement('script'); ga.type = 'text/javascript'; ga.async = true;
    ga.src = ('https:' == document.location.protocol ? 'https://ssl' : 'http://www') + '.google-analytics.com/ga.js';
    var s = document.getElementsByTagName('script')[0]; s.parentNode.insertBefore(ga, s);
  })();
</script>
<!-- End of Google Website Optimizer Tracking Script -->

Your account identifier is the string set in this line:

_gaq.push(['gwo._setAccount', 'UA-25983867-2']);

In this example, the account is 'UA-2598386702'

The experiment's id is the string set in this line:

_gaq.push(['gwo._trackPageview', '/0156495775/test']);

In this example, your experiment id would be anything before /test, so for 0156495775/test, the experiment id is 0156495775

Add your tags

On the control page, you want to add the following tag after the <head>

<gwo:include control='true' account='UA-25983867-2' experiment='0156495775'/>

On your variation pages, you want to include the following tags:

<gwo:include variation='true' account='UA-25983867-2' experiment='0156495775'/>

And on your conversion pages, you want the following:

<gwo:conversion account='UA-25983867-2' experiment='0156495775'/>

Validate your pages.

At this point, your AB test is set and ready to track. You can validate your pages by saving a local copy of your application page's HTML source.

Once your pages are validated, save your AB test and google website optimizer will begin serving alternatives whenever your visitors come to the original page.

2.2. Setting up a Multi-variate Test

This type of tests work better with scenarios where there are pieces of code that you wish to change within the same page, or changes scattered around several pages.

In Grails terms, these work best when you have to make changes to things that would normally go into templates.

Set up an experiment

  • Login to your GWO account and select Experiments > Create another experiment
  • Choose Multivariate Experiment - The most robust way to test lots of ideas
  • Run through the checklist and click on "I've completed the steps above checkbox"
  • Click on "Create"
  • Fill in all the fields in the page
  • Click on "Continue"
  • Select 'You will install and validate the JavaScript tags'

Get your account and experiment ids.

This procedure is the same as the one for A/B tests above except the control and tracking scripts are now merged into one on the page.

Add your tags

Add tracking tags

On your main testing page, you want to set an include tag with the multi section set to true

<gwo:include account='UA-25983867-2' experiment='0156495775'/>

And on your conversion pages, you want the following:

<gwo:conversion account='UA-25983867-2' experiment='0156495775'/>

Define sections

What makes a multivariate test different than a standard A/B test is that GWO allows you to define alternative content for individual sections on the page. You then create combinations of these alternatives within your tests to see which ones perform best.

To define a section to be tested, simply give it a name and surround it with the gwo:section tag. For example

<gwo:section name='bookingForm'>
   Booking Form
   <g:include controller="booking" action="make"/>
</gwo:section>

Validate your pages.

You can validate your pages by saving a local copy of your application page's HTML source.

At this point, you have identified the sections on your page you wish modify for multivariate testing.

You can use the tools provided by Website optimizer to create and modify variations at this point.

Within GWO, you can now define alternatives for the different sections you wish to test. Please refer to one of the books pointed in the Further reading section for how to use this feature.
